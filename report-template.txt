================================================================================

  版本: 1.0 | 创建时间: 2026-04-09
================================================================================

使用说明：
  - 每位 Agent 独立负责一个单元（.txt 文件）的生成工作。
  - 工作完成后，复制本模板并按以下格式填写，提交至 report.txt 对应条目，
    并将最终得分一并记录到 workflow.txt。
  - 填写时请删去「说明/示例」括号内的提示文字，保留结构标签。
  - 状态标记：⬜ 未开始 | 🔄 进行中 | ✅ 已完成 | ⏸ 暂停/阻塞

以下清单来源于 rule.txt 第七节「提交检查清单」，Agent 提交前必须全部勾选：

内容准确性：
  [ ] 所有 \run / \code 块内的代码已验证可编译（或注明"片段/伪代码"）
  [ ] 无事实性技术错误（函数签名、版本号、语义描述均准确）
  [ ] 超纲内容仅"提及+展望"，未展开

教学结构：
  [ ] 有"简介"节，建立学习动机
  [ ] 有"知识点列表"节，使用 \list{} 概览
  [ ] 有"知识点详解"节，知识点用 // 子标题分隔
  [ ] 有"难点易错点分析"节，包含至少1个 \danger{} 区块
  [ ] 有"实战作业"节，包含至少1道基础题

交互与标记：
  [ ] 所有交互标记语法正确（括号配对、必需参数齐全）
  [ ] 无发明新标记（只使用 rule.txt 第八节定义的标记）
  [ ] 交互元素数量符合对应难度等级的密度要求

写作风格：
  [ ] 技术术语首次出现标注中英双语
  [ ] 没有使用"显然""众所周知"等跳步词
  [ ] 语气符合章节难度等级（入门/中级/高级）

可选推荐项：
  [ ] 保留了"新版特性"节（如适用）
  [ ] 保留了"工程实践"节（如适用）

报告：
  [ ] report.txt 对应条目已完整填写（状态、交互清单、作业清单、备注）
  [ ] workflow.txt 已追加本文件的得分记录
