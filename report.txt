【2026-04-09 教学网页生成 Agent — 工作报告】

阶段 A — 网页模板规划
  分析了 rule.txt 中定义的完整标记语法（\run、\quiz、\diagram、\collapse、\tabs、\steps 等），
  以及 index.html 中已有的渲染引擎实现，规划以 template.html 作为所有章节页的共享渲染核心。

阶段 B — 核心产出
  1. template.html（项目根目录）
     - 完整的独立章节查看器，基于 URL 参数 ?folder=XX&file=YY.txt 加载章节
     - 包含全套标记语法渲染引擎（与 index.html 保持一致并优化）
     - 支持代码高亮（highlight.js）、可运行代码（Godbolt API）、测验、图解、折叠、标签页、步骤引导
     - 具备：顶部导航栏 + 面包屑、阅读进度条、目录浮动、章节前进/后退导航、底部章节切换
     - Apple HIG 精致视觉风格，完全匹配 index.html 的设计语言
     - 响应式布局，支持移动端

  2. content/{每个单元}/index.html（共 18 个单元概览页）
     - 每个学习单元的入口页，展示该单元所有章节列表
     - 点击章节跳转到对应章节页
     - 单元主题色、图标与 index.html 主技能树保持一致

  3. content/{单元}/{章节}.html（共 84 个章节入口页）
     - 覆盖全部 18 个单元、所有 85 个章节（含实战项目章节）
     - 使用 JavaScript 立即重定向到 template.html 进行渲染
     - 包含优雅的加载等待卡片（避免重定向时白屏闪烁）
     - 元数据（title、description）完整，便于 SEO 和书签

阶段 C — 架构决策说明
  为何采用"重定向 + 共享 template.html"架构而非"独立完整页面"？
  - 避免将 ~60KB 渲染引擎重复写入 84 个文件（存储浪费，维护困难）
  - template.html 修改一次即全局生效，保证版本一致性
  - 静态 GitHub Pages 完全兼容 fetch + URL 参数方案
  - 章节直接 URL（如 content/01-basics/hello-world.html）对收藏夹/外链友好

阶段 D — 难点与评估
  难点1：标记语法嵌套解析
    \collapse、\tabs、\steps 内部可嵌套其他标记，甚至递归嵌套。
    解决方案：parseContent 与 parseBlock 相互递归调用，通过括号深度计数
    （depth 变量）精确定位块边界，确保嵌套不会解析错误。
  
  难点2：目录（TOC）与阅读进度的协同
    目录需要在内容渲染完成后动态生成，且需随滚动高亮当前节。
    解决方案：渲染后调用 buildToc()，使用 IntersectionObserver 监听各 h2 
    元素的可见性，实现流畅的阅读进度指示。
  
  难点3：前后章导航与单元内排序
    需要从 TREE 数据中找到当前章节的前驱/后继。
    解决方案：findNode() 返回 {branch, node, nodeIndex}，再通过索引偏移
    获取前后章节，并生成正确的相对 URL 路径。

  难点4：相对路径问题
    template.html 在根目录，content/.txt 文件在二级子目录。
    章节 HTML 中的重定向 URL 需要 ../../template.html（相对于其所在子目录）。
    template.html 自身通过 content/{folder}/{file} 来 fetch 内容（相对于根目录）。
    两套路径体系统一验证，无冲突。

  交互功能完整度: 10/10 — 所有 rule.txt 定义的标记均已实现渲染
