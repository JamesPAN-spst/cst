/// 简介

本项目综合运用并发编程板块的全部知识，构建一个**游戏引擎风格的多线程任务调度系统（Job System）**。你将实现线程池、任务依赖图、帧同步屏障和性能追踪——这是现代游戏引擎（Unreal、Unity DOTS、自研引擎）的核心基础设施。

/// 项目需求

**项目名称：TaskForge — 多线程任务调度引擎**

实现一个支持任务依赖、优先级调度和帧同步的任务调度器，包含以下核心组件：

\list{
- **ThreadPool**：固定数量的工作线程，从优先级队列中取任务执行
- **Task / TaskId**：任务单元，包含工作函数、优先级、依赖列表
- **TaskGraph**：有向无环图（DAG）调度，依赖满足后自动入队
- **FrameScheduler**：按阶段（Physics→AI→Animation→Render）执行，每阶段用 barrier 同步
- **PerformanceTracer**：记录每个任务的线程ID、开始/结束时间，输出文本时间线
}

架构示意：

\diagram[flow]{
FrameScheduler -> TaskGraph -> ThreadPool
TaskGraph: 解析依赖 -> 就绪任务入队
ThreadPool: Worker 取任务 -> 执行 -> 通知后续任务
PerformanceTracer: 记录每个任务的时间线
帧同步: barrier 等待本阶段所有任务完成
}

/// 技术要求

\table{
C++ 特性 | 使用场景
---
`std::jthread` + `stop_token` | 工作线程（支持优雅关闭）
`std::mutex` + `lock_guard` | 任务队列保护
`std::condition_variable` | 工作线程等待新任务
`std::atomic<int>` | 任务依赖计数（waitCount）
`std::barrier` | 帧阶段同步点
`std::latch` | waitAll 等待所有任务完成
`alignas(64)` | 避免 false sharing（原子计数器）
`thread_local` | 每线程性能计数器
`std::chrono` | 高精度时间测量
}

        // 提交本阶段的所有任务到线程池

**4. PerformanceTracer（RAII 追踪）**

\code[cpp]{
class ScopedTrace {
    PerformanceTracer& tracer_;
    TraceEvent event_;
public:
    ScopedTrace(PerformanceTracer& t, std::string_view name)
        : tracer_(t), event_{std::string(name), threadId(), Clock::now(), {}} {}
    ~ScopedTrace() {
        event_.end = Clock::now();
        tracer_.addEvent(std::move(event_));
    }
};

/// 测试场景

以下测试场景必须全部通过：

\code[cpp]{
// 测试1：基础线程池
void testPool() {
    ThreadPool pool(4);
    std::atomic<int> sum{0};
    for (int i = 0; i < 100; ++i)
        pool.submit([&sum]{ sum.fetch_add(1); });
    pool.waitIdle();
    assert(sum == 100);
}

// 测试2：任务依赖图
void testGraph() {
    ThreadPool pool(4);
    TaskGraph graph;
    std::vector<int> order;
    std::mutex mtx;

    auto a = graph.createTask([&]{ lock_guard l(mtx); order.push_back(1); }, "A");
    auto b = graph.createTask([&]{ lock_guard l(mtx); order.push_back(2); }, "B");
    auto c = graph.createTask([&]{ lock_guard l(mtx); order.push_back(3); }, "C");
    graph.addDependency(c, a); // C 依赖 A
    graph.addDependency(c, b); // C 依赖 B
    assert(!graph.hasCycle());

    graph.submit(pool);
    graph.waitAll();
    assert(order.back() == 3); // C 一定最后执行
}

// 测试3：帧调度（10 帧压力测试）
void testFrameScheduler() {
    ThreadPool pool(4);
    FrameScheduler sched(pool);
    for (int f = 0; f < 10; ++f) {
        sched.schedule(Phase::Physics, [f]{ /* 物理 */ });
        sched.schedule(Phase::AI,      [f]{ /* AI */   });
        sched.schedule(Phase::Render,  [f]{ /* 渲染 */ });
        sched.executeFrame(); // 阶段间 barrier 同步
    }
}

/// 扩展挑战

\list{
- 实现**工作窃取调度**：每个 worker 有本地队列，空闲时从其他 worker 偷取任务
- 支持**任务取消**：取消一个任务时自动取消所有依赖它的后续任务
- 实现 **Chrome Trace JSON** 导出，在 `chrome://tracing` 中可视化任务时间线
- 添加**内存分配追踪**：每个 worker 使用 thread_local 帧分配器，帧结束时批量释放
- 实现 **fiber/协程版本**：任务可以 `co_await` 依赖而非阻塞线程
- 集成前面章节的**自定义内存分配器**，每个 worker 独立内存池
}
