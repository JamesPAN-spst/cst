/// 简介

上节课我们学了函数——把代码打包成可复用的小工具。但你有没有想过：函数之间怎么**共享同一份数据**？拷贝一份传过去？如果是个巨大的对象呢，每次拷贝岂不是慢死？

这就是指针（Pointer）和引用（Reference）登场的理由。

**指针**就像一张**写着门牌号的小纸条**。它不是东西本身，而是告诉你"东西放在哪里"。有了纸条，你可以按地址找到那个值，直接在原地修改——不用搬来搬去。

**引用**更简单：它就是给变量起了个**外号**。你叫一个同学"小明"，也叫他"阿明"——不管叫哪个名字，指的都是同一个人。

听起来挺简单？坑可不少。C++ 里一半以上的 bug 都跟指针有关：空指针（Null Pointer）崩溃、悬空指针（Dangling Pointer）读到垃圾值、内存泄漏（Memory Leak）……别怕，这节课我们一个个拆开讲清楚。

/// 知识点详解

// 指针基础——小纸条和门牌号

先看一个最简单的变量：

\code[cpp]{
int x = 42;
}

`x` 是一个盒子，里面放着 `42`。这个盒子在内存里有一个**地址**（就像你家有门牌号）。指针，就是**专门记录这个门牌号的变量**。

\code[cpp]{
int x = 42;
int* p = &x;   // & 取地址：拿到x的门牌号，写到小纸条p上
}

拆开看：
- `int*` = "这是一张纸条，上面写的地址指向一个 `int` 盒子"
- `&x` = "取 `x` 的地址"（`&` 在这里念"取地址"）
- `*p` = **解引用（Dereference）**：按照纸条上的地址，找到那个盒子，读/改里面的值

**类比：** 你手里有张纸条写着"XX路42号"。`&x` 就是查这个盒子的地址，`*p` 就是按地址找到那户人家敲门。

来，动手看看指针到底怎么工作——点"▶ 运行"：

\run[cpp]{
#include <iostream>
using namespace std;

int main() {
    int x = 42;
    int* p = &x;      // p存了x的地址

    cout << "x 的值:  " << x << endl;
    cout << "x 的地址: " << &x << endl;
    cout << "p 的值（即x的地址）: " << p << endl;
    cout << "p 指向的值（*p）: " << *p << endl;

    *p = 100;          // 通过小纸条找到x，把值改成100
    cout << "修改后 x = " << x << endl;  // x也变了！

    return 0;
}
}

看到了吗？通过 `*p = 100`，我们没有直接碰 `x`，但 `x` 的值真的变了！因为 `p` 指向 `x`，修改 `*p` 就是在修改 `x` 本身。

下面这张图展示了上面代码在内存中的样子：

\diagram[memory]{
stack:
  [0x7ff4] int x = 42
  [0x7fe8] int* p = 0x7ff4
arrow: p -> x
}

`p` 里存的不是 `42`，而是 `0x7ff4`——那是 `x` 的地址。箭头表示"p 指向 x"。

\tip{声明指针时建议 `int* p` 风格（星号跟类型），但**每行只声明一个变量**。因为 `int* a, b;` 里 `a` 是指针、`b` 却是普通 `int`！这个坑无数人踩过。}

// 指向指针的指针

指针本身也是个变量，也有地址。所以你可以用另一个指针指向它：

\code[cpp]{
int x = 42;
int* p = &x;     // p 指向 x
int** pp = &p;   // pp 指向 p（指向指针的指针）

**pp = 200;      // 先找到p，再找到x，把x改成200
}

**类比：** 你问同学A"小明家在哪？"，A说"去问B"。B才知道小明家地址。`pp` → `p` → `x`，两层间接。

初学阶段你不太会用到 `int**`，但理解它存在就好。后面学动态数组或函数指针参数时会再遇到。

// 空指针：nullptr——"地址不存在"

有时候你想表示"这张纸条上还没写地址"或者"这个指针不指向任何东西"。这就是**空指针（Null Pointer）**。

C++ 有三种写法，但只有一种是对的：

\code[cpp]{
int* p1 = nullptr;  // ✅ C++11 正确写法
int* p2 = NULL;     // ❌ 老写法，实际上就是数字 0
int* p3 = 0;        // ❌ 更老的写法
}

为什么 `NULL` 和 `0` 不行？看这个**先错后对**的例子：

\code[cpp]{
// 假设你写了两个同名函数（重载）
void process(int n)   { cout << "处理数字" << endl; }
void process(int* p)  { cout << "处理指针" << endl; }

process(0);        // 调用了 process(int)！你本来想表示空指针啊！
process(NULL);     // 也可能调用 process(int)！因为NULL常常就是0
process(nullptr);  // ✅ 一定调用 process(int*)！
}

`nullptr` 的类型是 `std::nullptr_t`，编译器知道它是指针，不是数字。**永远用 `nullptr`，忘掉 `NULL` 和 `0`。**

\danger{**解引用空指针 = 程序必崩！** 就像按一个不存在的门牌号去找人，操作系统直接给你报错（段错误，Segmentation Fault）。这是最常见的 C++ 崩溃原因之一。使用指针前一定要检查：`if (p != nullptr) { *p = 42; }`}

// 指针算术——在一排房子间跳跳跳

如果内存里有一排连续的 `int` 盒子（比如数组），指针可以在它们之间跳来跳去：

\code[cpp]{
int arr[] = {10, 20, 30, 40, 50};
int* p = arr;      // 数组名自动变成"指向第一个元素的指针"

// p + 0 → arr[0]，值10
// p + 1 → arr[1]，值20（向后跳一个int的距离，不是一个字节！）
// p + 2 → arr[2]，值30

cout << *(p + 2) << endl;  // 30
cout << p[2] << endl;      // 也是30！p[2] 等价于 *(p + 2)
}

**类比：** 想象一条街上有5栋连排别墅，门牌号分别是1号、2号……5号。你站在1号门口（`p` 指向 `arr[0]`），往前走2步就到了3号（`p + 2` 指向 `arr[2]`）。注意每一步跨的距离取决于盒子的大小——`int` 一步跨4字节，`double` 一步跨8字节。

\danger{**指针算术只在同一个数组内部合法！** 对两个不相关变量的指针做加减或比较是**未定义行为（Undefined Behavior，UB）**——编译器不报错，但运行结果可能是任何东西。}

// const 与指针的组合——到底谁不能改？

`const` 和指针组合时新手经常分不清。记住一个**口诀**：

**`const` 在 `*` 左边 → 值不可改；`const` 在 `*` 右边 → 指针不可改**

\steps{
\step[指向 const 的指针]{
\code[cpp]{
int x = 10, y = 20;
const int* p = &x;  // const 在 * 左边 → 不能通过p改值

*p = 99;   // ❌ 错误！值被锁定了
p = &y;    // ✅ 可以！纸条本身可以换个地址写
}

**类比：** 你有一张纸条，上面的地址可以擦了重写（换指向），但是找到那户人家后**不许动人家的东西**。
}
\step[const 指针（指针本身不可改）]{
\code[cpp]{
int x = 10, y = 20;
int* const p = &x;  // const 在 * 右边 → 纸条不能换地址

*p = 99;   // ✅ 可以！找到人家后随便改
p = &y;    // ❌ 错误！纸条被钉死了
}

**类比：** 纸条用胶水粘在墙上了，永远指向 `x`。但你可以通过它修改 `x` 的值。
}
\step[双 const（都不能改）]{
\code[cpp]{
int x = 10;
const int* const p = &x;  // 左右都有 const

*p = 99;   // ❌ 不能改值
p = &y;    // ❌ 也不能换地址
}

最严格的组合：纸条粘死了，而且不许改目标的东西。
}
}

\tip{**从右往左读**也可以帮你理解：`const int* p` → "p 是一个 pointer，指向 int，int 是 const"。`int* const p` → "p 是一个 const pointer，指向 int"。}

// 引用——变量的"外号"

引用（Reference）比指针简单多了。它就是给一个已存在的变量**起个别名**：

\code[cpp]{
int x = 42;
int& ref = x;   // ref 是 x 的外号（别名）
ref = 100;       // 改ref就是改x！
cout << x;       // 输出100
}

**类比：** 你叫"小明"，同学给你起了个外号叫"阿明"。不管叫"小明"还是"阿明"，指的都是你这个人。改了"阿明"的成绩，"小明"的成绩也变了——因为是同一个人嘛。

来，动手试试引用的威力：

\run[cpp]{
#include <iostream>
using namespace std;

// 不用引用：函数拿到的是拷贝，改了也白改
void addTenCopy(int n) {
    n += 10;  // 改的是拷贝，原变量不受影响
}

// 用引用：函数拿到的是"外号"，改它就是改原变量
void addTenRef(int& n) {
    n += 10;  // 直接改原变量！
}

int main() {
    int score = 80;

    addTenCopy(score);
    cout << "拷贝版之后: " << score << endl;  // 还是80！

    addTenRef(score);
    cout << "引用版之后: " << score << endl;  // 变成90了！

    return 0;
}
}

这就是引用最常见的用法——**让函数能修改调用者的变量**。不用引用，函数只能操作拷贝，改了也白改。

引用有几条铁规矩：

\code[cpp]{
int x = 42, y = 99;

int& ref = x;   // ✅ 引用必须在创建时就绑定
// int& bad;     // ❌ 错误！引用不能"空着"不绑定

ref = y;         // ⚠ 这不是让ref改指向y！这是把y的值赋给x！
                 // ref依然是x的外号，x现在变成了99

// 一句话：引用一旦绑定，终身不变。
}

\note{`const` 引用有个特殊能力——可以绑定到临时值（右值）：`const int& r = 42;` 是合法的。编译器会延长这个临时值的生命周期，直到 `r` 销毁。这在函数传参时特别有用：`void print(const string& s)` 可以同时接受变量和字面量。}

下面这张图对比了指针和引用在内存中的区别：

\diagram[memory]{
stack:
  [0x7ff4] int x = 42
  [0x7ff0] int* p = 0x7ff4
  [0x7fec] int& ref → (x的别名，不占独立空间)
arrow: p -> x
arrow: ref -> x
}

概念上引用不是独立的对象（它就是 `x` 的另一个名字），但编译器内部通常用指针来实现引用。你不需要关心这个实现细节，只要记住：**引用 = 别名，创建就绑定，不能改绑**。

// 左值引用 vs 右值引用（概述）

到目前为止我们用的 `int&` 叫**左值引用（Lvalue Reference）**——给一个"有名字、有地址"的变量取外号。

C++11 引入了**右值引用（Rvalue Reference）**，写法是 `int&&`（两个 `&`）：

\code[cpp]{
int x = 42;

int& lref = x;       // ✅ 左值引用：绑定到左值（有名字的东西）
// int& bad = 42;     // ❌ 左值引用不能绑定到临时值42

int&& rref = 42;     // ✅ 右值引用：绑定到右值（临时的、即将消失的东西）
// int&& bad2 = x;    // ❌ 右值引用不能直接绑定到左值
int&& rref2 = std::move(x);  // ✅ std::move把左值"伪装"成右值
}

右值引用是**移动语义（Move Semantics）**的基础——把即将销毁的对象的资源"偷过来"而不是拷贝，性能飞升。但这是进阶话题，我们在"04-现代 C++ 特性 → 右值引用与移动"章节再展开。

\collapse[🔮 预览：智能指针——指针的终极进化]{
裸指针（`int* p = new int(42);`）的最大问题是：**谁负责 `delete`？** 忘了就内存泄漏，多 `delete` 一次就崩溃。

C++11 引入了**智能指针（Smart Pointer）**，自动管理内存生命周期：
- `std::unique_ptr<T>` — 独占所有权，出了作用域自动释放
- `std::shared_ptr<T>` — 多个指针共享同一资源，最后一个销毁时释放
- `std::weak_ptr<T>` — 辅助 `shared_ptr`，打破循环引用

\code[cpp]{
#include <memory>
auto p = std::make_unique<int>(42);  // 自动管理，不用手写delete
// p 离开作用域时，内存自动释放
}

现代 C++ 的原则是：**应用层代码几乎不应该出现裸 `new` / `delete`**。智能指针的详细用法见"04-现代 C++ → 智能指针"章节。
}

// 悬空指针与悬空引用——人搬走了，纸条还在

**最凶残的 bug 来了。** 所谓悬空（Dangling），就是指针/引用指向的对象已经不在了，但你还傻乎乎地去读写。

**类比：** 你手里的纸条写着"XX路42号"，你以为小明还住那里。可小明上个月搬走了，42号现在住着一位陌生人。你按地址去找"小明"——找到的是别人的数据。

\run[cpp]{
#include <iostream>
using namespace std;

int* createDangling() {
    int local = 42;        // local 是函数内部的局部变量
    return &local;         // ⚠ 返回局部变量的地址！
}   // local 在这里被销毁，地址变成"已搬走"

int main() {
    int* p = createDangling();
    // p 现在是悬空指针！local 已经不存在了
    // 下面这行可能输出42，也可能输出垃圾值，也可能崩溃
    // 这就是未定义行为的恐怖之处
    cout << "悬空指针的值: " << *p << endl;

    return 0;
}
}

\danger{**Use-after-free 是 C++ 中最严重的安全漏洞之一！** 攻击者可以利用悬空指针读取敏感数据甚至执行恶意代码。养成习惯：`delete` 后立即将指针置为 `nullptr`；优先使用智能指针；注意容器操作后迭代器/引用可能失效。}

另一个隐蔽的悬空场景——容器扩容导致引用失效：

\code[cpp]{
#include <vector>

std::vector<int> v = {1, 2, 3};
int& ref = v[0];     // ref 绑定到第一个元素
v.push_back(4);      // vector可能重新分配内存，搬到新地址！
// ref 现在可能悬空！旧地址已经被释放了
}

// 指针 vs 引用——什么时候用哪个？

\tabs{
\tab[选引用的场景]{
- **函数传参**：`void process(const string& s)` — 不拷贝、不修改
- **函数返回**：`int& operator[](int i)` — 返回容器元素的引用，允许 `v[0] = 10`
- **给变量取别名**：简化复杂表达式 `auto& item = map["key"]`
- **永远不可能为空**：引用必须绑定，所以不需要判空

**原则：能用引用就用引用，语法简洁、安全。**
}
\tab[只能用指针的场景]{
- **可能为空**：表示"可能没有"的语义（`TreeNode* left = nullptr;`）
- **需要重新指向**：引用绑了就不能改，指针可以随时换
- **动态内存**：`new` 返回的是指针
- **数组遍历**：指针算术遍历 C 风格数组
- **与 C 语言交互**：C 没有引用，接口只能用指针

**原则：用指针时一定要管好生命周期，能用智能指针就不用裸指针。**
}
}

/// 难点易错点分析

\danger{**`int* a, b;` 声明了一个指针和一个普通 `int`！** 星号是跟变量名而不是跟类型走的。要声明两个指针，写 `int *a, *b;` 或者分两行写。最佳实践：**一行只声明一个变量**。}

\danger{**数组退化（Array Decay）：** 数组在绝大多数表达式中会自动变成指向首元素的指针，**丢失大小信息**：}

\code[cpp]{
int arr[5] = {1, 2, 3, 4, 5};
auto p = arr;        // p 是 int*，不是 int[5]！
sizeof(arr);         // 20（5个int × 4字节）
sizeof(p);           // 8（64位系统上指针大小）—— 大小信息丢了！

// 只有三种情况数组不退化：
// 1. sizeof(arr)
// 2. &arr（得到 int(*)[5]，指向整个数组的指针）
// 3. decltype(arr)（推断出 int[5]）
}

\danger{**引用不能重新绑定：** `ref = y` 不是让 ref 绑到 y，而是把 y 的值赋给 ref 当前绑定的变量！这是新手最容易误会的地方。}

\danger{**解引用空指针 = 必崩。** 使用前一定要检查 `if (p != nullptr)` 或者更好的做法——用智能指针避免手动管理。}

现在来检验一下你的理解——下面几道题做做看：

\quiz[choice]{
Q: 下面哪行代码**编译报错**？
\code[cpp]{
int x = 10, y = 20;
const int* p = &x;    // 第1行
*p = 99;               // 第2行
p = &y;                // 第3行
}
A: 第1行
B: 第2行
C: 第3行
D: 都不报错
ANS: B
EXP: `const int* p` 表示"不能通过 p 修改指向的值"。第2行试图通过 `*p` 修改值，编译失败。第3行改变 p 的指向是允许的，因为 `const` 在 `*` 左边只锁值，不锁指针本身。
}

\quiz[tf]{
Q: `int& ref;`（不初始化的引用声明）是合法的 C++ 代码。
ANS: F
EXP: 引用**必须在声明时初始化**，不能留空。这和指针不同——指针可以先声明为 `nullptr` 再赋值，但引用不行。
}

\quiz[choice]{
Q: 以下代码执行后，`x` 的值是多少？
\code[cpp]{
int x = 10;
int& ref = x;
int y = 20;
ref = y;
}
A: 10
B: 20
C: 未定义行为
D: 编译错误
ANS: B
EXP: `ref = y` 不是让 ref 绑定到 y！ref 永远是 x 的别名。`ref = y` 实际上是 `x = y`，把 y 的值 20 赋给 x。所以 x 变成了 20。
}

/// 实战作业

🟢 **基础题1：指针交换两个变量**

不借助引用，用**指针**写一个函数 `swapByPtr`，交换两个 `int` 变量的值。补全 `/* TODO */` 部分：

\run[cpp]{
#include <iostream>
using namespace std;

void swapByPtr(int* a, int* b) {
    /* TODO: 用指针交换 *a 和 *b 的值（提示：需要一个临时变量） */
}

int main() {
    int x = 10, y = 20;
    cout << "交换前: x=" << x << ", y=" << y << endl;

    swapByPtr(&x, &y);

    cout << "交换后: x=" << x << ", y=" << y << endl;
    // 期望输出: x=20, y=10

    return 0;
}
}

🟢 **基础题2：引用版本对比**

这次用**引用**实现同样的交换功能。注意调用方式的区别——不需要取地址符号 `&`：

\run[cpp]{
#include <iostream>
using namespace std;

void swapByRef(int& a, int& b) {
    /* TODO: 交换 a 和 b 的值 */
}

int main() {
    int x = 10, y = 20;
    cout << "交换前: x=" << x << ", y=" << y << endl;

    swapByRef(x, y);  // 注意：不用写 &x, &y

    cout << "交换后: x=" << x << ", y=" << y << endl;
    // 期望输出: x=20, y=10

    return 0;
}
}

🟡 **进阶题：找数组最大值并返回其指针**

写一个函数，接收一个 `int` 数组和长度，返回指向最大元素的指针。调用者可以通过返回的指针修改最大元素的值。

\collapse[🤔 参考思路]{
1. 用一个 `int* maxPtr` 先指向数组第一个元素
2. 遍历数组，如果发现比 `*maxPtr` 更大的元素，更新 `maxPtr`
3. 返回 `maxPtr`

\code[cpp]{
int* findMax(int* arr, int size) {
    int* maxPtr = arr;
    for (int i = 1; i < size; i++) {
        if (arr[i] > *maxPtr) {
            maxPtr = &arr[i];  // 或者 maxPtr = arr + i;
        }
    }
    return maxPtr;  // 安全：arr的生命周期比函数长
}
}

注意这里返回指针是安全的——`arr` 是调用者的数组，不会在函数返回后消失。和悬空指针的区别在于：我们**没有返回局部变量的地址**。
}
