/// 简介

上节课我们学了变量和数据类型——知道怎么"装东西"了。但光装不行啊，你得**操作**这些数据才有用对吧？运算符（Operator）就是你操作数据的"动词"。

`+` `-` `*` `/` 这些你小学就见过了——但C++的运算符远不止这些。比较大小、逻辑判断、位操作、甚至改变数据类型，全靠运算符。搞清楚每种运算符怎么用，以及它们的**优先级**（谁先算谁后算），是写出正确代码的基本功。

这节课结束后，你就能用运算符组合出复杂的表达式，而不会掉进"整数除法""赋值vs比较"这些经典坑。

/// 知识点列表

\list{
- 算术运算符（Arithmetic Operators）：+ - * / %
- 赋值运算符（Assignment Operators）：= += -= *= /= %=
- 比较运算符（Comparison Operators）：== != < > <= >=
- 逻辑运算符（Logical Operators）：&& || !
- 自增自减运算符（Increment/Decrement）：++ --
- 类型转换（Type Casting）
- 运算符优先级（Operator Precedence）
}

/// 知识点详解

// 算术运算符

算术运算符就是你数学课上的老朋友。加减乘除取余——但在C++里，**除法有个大坑**。

先看一段"理所当然"但会出错的代码：

\code[cpp]{
int a = 10, b = 3;

cout << a + b << endl;   // 13   加法，没毛病
cout << a - b << endl;   // 7    减法，没毛病
cout << a * b << endl;   // 30   乘法，没毛病
cout << a / b << endl;   // 3    ← 等等！不是3.33吗？？
cout << a % b << endl;   // 1    取模（余数：10÷3=3余1）
}

\danger{**整数除法会直接丢弃小数部分！** `10 / 3 = 3`（不是3.33）。这是新手被炸得最惨的地方。两个 `int` 做除法，结果还是 `int`，小数直接没了。}

那怎么拿到正确的小数结果呢？——至少让一个操作数变成浮点数：

\code[cpp]{
cout << 10.0 / 3 << endl;    // 3.33333   ← double ÷ int = double
cout << 10 / 3.0 << endl;    // 3.33333   ← int ÷ double = double
cout << (double)10 / 3 << endl; // 3.33333 ← 强制转成double再除
}

\tip{**取模运算符 `%` 只能用于整数。** `10.5 % 3` 会直接编译报错！取模在很多场景超有用：判断奇偶（`n % 2 == 0` 就是偶数）、循环归零（计数到N就重置）、游戏逻辑（回合制谁先谁后）。}

来动手试试，改改数字看看整数除法和浮点除法的区别：

\run[cpp]{
#include <iostream>
using namespace std;

int main() {
    int a = 17, b = 5;

    // 整数除法 —— 直接丢掉小数
    cout << "整数除法: " << a << " / " << b << " = " << a / b << endl;
    cout << "取模(余数): " << a << " % " << b << " = " << a % b << endl;

    // 浮点除法 —— 保留小数
    cout << "浮点除法: " << (double)a / b << endl;

    // 试试：把 a 改成 7，b 改成 2，输出是什么？
    // 再试：a = -7, b = 2，整数除法和取模的结果是什么？
    return 0;
}
}

\collapse[🤔 负数取模是怎么回事？]{
在C++11之后，**整数除法向零截断**（truncation toward zero），取模的符号跟被除数一致：

`-7 / 2 = -3`（不是-4）
`-7 % 2 = -1`（因为 -7 = (-3)*2 + (-1)）

这个规则初学者不用刻意记，但如果你发现取模结果出了负数，就回来看看这段。
}

// 赋值运算符

赋值运算符 `=` 是"把右边的值**放进**左边的变量"，**不是**数学里的"等于"。

\code[cpp]{
int x = 10;    // 把10放进x

x += 5;        // 等价于 x = x + 5   → x是15
x -= 3;        // 等价于 x = x - 3   → x是12
x *= 2;        // 等价于 x = x * 2   → x是24
x /= 4;        // 等价于 x = x / 4   → x是6
x %= 5;        // 等价于 x = x % 5   → x是1
}

\tip{复合赋值运算符（`+=` `-=` `*=` `/=` `%=`）就是偷懒写法。`x += 5` 和 `x = x + 5` 完全等价，但前者更短更清爽，也是老手们的惯用写法。}

\danger{**`=` 和 `==` 完全不同！** 这是C++新手犯错率最高的地方之一。
- `=` 是**赋值**：把右边的值塞进左边的变量
- `==` 是**比较**：判断两边是否相等，返回 true/false

经典bug长这样：`if (x = 5)` ← 这不是在比较x和5！这是把5赋值给x，然后判断5是不是true（当然是）。正确写法是 `if (x == 5)`。}

// 比较运算符

比较运算符（Comparison Operators）用来比大小、判相等，结果是 `bool` 类型（`true` 或 `false`）。

\code[cpp]{
int a = 10, b = 20;

cout << (a == b) << endl;  // 0 (false)  相等？
cout << (a != b) << endl;  // 1 (true)   不等？
cout << (a < b) << endl;   // 1 (true)   小于？
cout << (a > b) << endl;   // 0 (false)  大于？
cout << (a <= b) << endl;  // 1 (true)   小于等于？
cout << (a >= b) << endl;  // 0 (false)  大于等于？
}

\note{**C++默认输出 true/false 时显示 1/0。** 如果你想看到文字"true"和"false"，在输出前加上 `cout << boolalpha;` 就行。}

// 逻辑运算符

逻辑运算符（Logical Operators）用来组合多个条件。想象你在游戏里进副本——不是光有武器就行，还得满足一堆条件：

\code[cpp]{
bool hasSword = true;
bool hasShield = false;
int level = 15;

// && = 且（两个都为true才行）
cout << (hasSword && hasShield) << endl;  // false —— 有剑但没盾，不行

// || = 或（至少一个为true就行）
cout << (hasSword || hasShield) << endl;  // true —— 有剑就够了

// ! = 非（取反）
cout << !hasShield << endl;               // true —— 没盾→true

// 组合判断
bool canEnterDungeon = (level >= 10) && (hasSword || hasShield);
// 等级>=10 且 (有剑或有盾) → true，可以进！
}

\diagram[flow]{
短路求值 (Short-Circuit Evaluation)

A && B:
  [计算A] --A为false--> [结果=false, B不执行]
  [计算A] --A为true--> [计算B] --> [结果=B的值]

A || B:
  [计算A] --A为true--> [结果=true, B不执行]
  [计算A] --A为false--> [计算B] --> [结果=B的值]
}

\tip{**短路求值（Short-Circuit Evaluation）非常重要：**
- `A && B`：如果A是false，B**直接跳过不执行**（因为不管B是啥，结果都是false）
- `A || B`：如果A是true，B**直接跳过不执行**（因为不管B是啥，结果都是true）

这在实际编程中经常利用——比如先判断指针不为空，再访问指针指向的值：`if (p != nullptr && p->value > 0)`。如果 `p` 是空指针，后面的 `p->value` 根本不会执行，避免了崩溃。}

// 自增和自减

`++` 和 `--` 是让变量"加1"和"减1"的快捷方式。但它们有**前置**和**后置**两种写法，在表达式中结果不一样！

先看简单用法——单独用的时候前置后置没区别：

\code[cpp]{
int i = 5;
i++;    // i变成6（后置自增）
++i;    // i变成7（前置自增）
i--;    // i变成6（后置自减）
--i;    // i变成5（前置自减）
}

放在表达式里就有区别了——先看一段"看似简单但容易翻车"的代码：

\code[cpp]{
int a = 5;
int b = a++;   // 后置：先把a当前值(5)给b，然后a加1 → b=5, a=6
int c = ++a;   // 前置：先给a加1(变成7)，然后把新值给c → c=7, a=7
}

\tabs{
\tab[前置 ++i]{
**先加后用。** 先把变量加1，再返回**新值**。

\code[cpp]{
int a = 5;
int b = ++a;  // a先变成6，然后b=6
// 此时 a=6, b=6
}
}
\tab[后置 i++]{
**先用后加。** 先返回变量的**旧值**，然后变量加1。

\code[cpp]{
int a = 5;
int b = a++;  // b先拿到5，然后a变成6
// 此时 a=6, b=5
}
}
}

\warning{**给初学者的忠告：不要把 `++` / `--` 塞进复杂表达式！** 像 `a = b++ + ++c` 这种代码，就算能编译也没人看得懂。老老实实分成几行写：先 `b++;`，再 `++c;`，再算赋值。代码是给人读的，不是给编译器炫技的。}

// 类型转换

有时候你需要把一种类型变成另一种。C++有两种方式：编译器偷偷帮你转（隐式），或者你自己明确说"给我转"（显式）。

先看**隐式转换（Implicit Conversion）**——编译器自动做的：

\code[cpp]{
int a = 10;
double b = a;       // int → double，安全，10变成10.0
int c = 3.99;       // double → int，小数部分直接没了！c = 3
}

\danger{**`double → int` 是直接截断（Truncation），不是四舍五入！** `(int)3.99 = 3`，`(int)-2.7 = -2`。如果你需要四舍五入，要用 `round()` 函数（需要 `#include <cmath>`）。}

再看**显式转换（Explicit Conversion / Type Casting）**——你主动告诉编译器要转：

\code[cpp]{
double x = 7.9;
int y = static_cast<int>(x);   // C++推荐写法，y = 7
int z = (int)x;                 // C风格转换（能用但不推荐）
}

\tip{**推荐用 `static_cast<目标类型>(值)` 做显式转换。** 虽然写起来长一点，但它让你的意图非常明确，而且编译器能帮你检查转换是否合理。C风格的 `(int)x` 太野蛮——它几乎什么转换都能做，包括一些危险的转换。}

\run[cpp]{
#include <iostream>
#include <cmath>
using namespace std;

int main() {
    double score = 89.7;

    // 看看不同方式的结果
    cout << "直接截断:   " << (int)score << endl;        // 89
    cout << "四舍五入:   " << round(score) << endl;      // 90
    cout << "static_cast: " << static_cast<int>(score) << endl; // 89

    // 整数除法 vs 浮点除法
    int total = 17, count = 5;
    cout << "整数除法:   " << total / count << endl;              // 3
    cout << "浮点除法:   " << static_cast<double>(total) / count << endl; // 3.4

    return 0;
}
}

// 运算符优先级

跟数学一样，C++中不同运算符有不同的**优先级（Precedence）**。先乘除后加减，括号最优先。

\code[cpp]{
int result = 2 + 3 * 4;    // = 2 + 12 = 14（先算*）
int result2 = (2 + 3) * 4; // = 5 * 4 = 20（括号优先）
}

\diagram[compare]{
left_title: 不加括号 — 容易出错
left:
  2 + 3 * 4
  = 2 + 12    ← 先算乘法
  = 14

right_title: 加括号 — 清晰明确
right:
  (2 + 3) * 4
  = 5 * 4     ← 先算括号
  = 20
}

\tip{**黄金法则：不确定优先级就加括号！** `(a + b) * c` 比 `a + b * c` 清晰一万倍。你写代码不是参加"记忆优先级大赛"——加个括号又不花钱，但能省掉无数debug时间。}

优先级从高到低的简化版参考：

\table{
优先级 | 运算符 | 说明
---
最高 | () | 括号
↓ | ++ -- ! | 自增、自减、逻辑非
↓ | * / % | 乘、除、取模
↓ | + - | 加、减
↓ | < > <= >= | 比较大小
↓ | == != | 相等/不等
↓ | && | 逻辑与
↓ | || | 逻辑或
最低 | = += -= *= /= | 赋值（包括复合赋值）
}

\collapse[📝 怎么记这张表？]{
其实你不用硬背这张表。记住三条就够了：
1. **括号最优先** —— 不确定就加括号
2. **算术 > 比较 > 逻辑 > 赋值** —— 先算数字，再比大小，再做逻辑，最后赋值
3. **`&&` 比 `||` 优先** —— 跟数学里"乘法比加法优先"的道理一样（AND像乘，OR像加）

实际写代码时，90%的优先级问题一个括号就解决了。
}

/// 难点易错点分析

\danger{**5个必须记住的坑（每个都是新手血泪史）：**

**1. 整数除法丢精度** → `7 / 2 = 3`，不是 `3.5`。解法：让至少一个操作数是 `double`。

**2. `=` 和 `==` 混淆** → `if(x = 5)` 是赋值，永远为true。应该写 `if(x == 5)`。

**3. 类型转换截断** → `(int)3.99 = 3`，不是 `4`。想四舍五入用 `round()`。

**4. 对浮点数取模** → `5.0 % 2` 编译报错！`%` 只能用于整数。浮点取余用 `fmod()`。

**5. 前置后置混着用** → `a = b++ + ++c` 这种代码，就算你写的时候清楚，三天后你自己也看不懂。分行写！}

\quiz[choice]{
Q: 以下代码输出什么？`int a = 10; int b = 3; cout << a / b;`
A: 3.33
B: 3
C: 4
D: 编译错误
ANS: B
EXP: 两个 `int` 做除法，结果还是 `int`，小数部分直接丢弃。10 ÷ 3 = 3 余 1，所以输出 3。想要 3.33... 需要至少一个操作数是浮点数。
}

\quiz[tf]{
Q: `if (x = 0)` 和 `if (x == 0)` 的效果完全一样。
ANS: F
EXP: `x = 0` 是赋值操作——它把 0 赋给 x，然后表达式的值是 0（即 false），所以 if 体永远不会执行。而 `x == 0` 是比较操作——只有当 x 确实等于 0 时才为 true。这是C++最经典的 bug 之一！
}

\quiz[choice]{
Q: `int a = 5; int b = a++; int c = ++a;` 执行后，a、b、c 的值分别是？
A: a=7, b=5, c=7
B: a=7, b=6, c=7
C: a=6, b=5, c=6
D: a=7, b=5, c=6
ANS: A
EXP: `b = a++`：后置自增，先把 a 当前值(5)给 b，再 a 加1→ b=5, a=6。`c = ++a`：前置自增，先给 a 加1(变成7)，再把新值给 c→ c=7, a=7。
}
