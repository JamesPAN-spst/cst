/// 简介

上节课我们学会了声明变量——给数据起名字、存数据。但你有没有想过：电脑怎么知道 `42` 是一个整数，`3.14` 是小数，`'A'` 是字母？

答案就是**数据类型（Data Type）**。就像你寄快递要选包裹大小一样——寄手机用小盒子，寄电视用大箱子——C++也需要你告诉它："这个变量要装什么类型的数据"，这样它才知道该分多大的内存"箱子"，以及怎么处理箱子里的东西。

学完这节课，你会清楚 C++ 里最常用的几种数据类型，知道什么时候该用哪种，还能避开一些新手必踩的坑（比如整数溢出和浮点精度问题）。

/// 知识点详解

// 为什么需要数据类型？

你去超市买东西，收银员需要知道每件商品的**价格**才能结账。同样，电脑处理数据时需要知道每个数据的**类型**，原因有三：

1. **不同类型占用不同大小的内存。** 存一个年龄（0~150）和存一本书的页数（可能上千页）需要的"箱子"可能一样大，但和存一个银河系的星星数量就完全不同了。
2. **不同类型的运算规则不同。** 整数除法 `7/2=3`（直接丢掉小数部分！），小数除法 `7.0/2.0=3.5`——同样是"除"，结果天差地别。
3. **编译器（Compiler）通过类型帮你抓错。** 如果你把一段文字塞进一个整数变量，编译器会拍桌子警告你。

\collapse[🤔 想一想：如果没有类型会怎么样？]{
想象一下，如果 C++ 不区分类型——所有数据都当一堆 0 和 1 随便混用。那写个 `7 / 2` 到底该给 `3` 还是 `3.5`？把成绩 `'A'` 做加法该加出什么？没有类型规则，程序世界就像一个没有交通灯的十字路口——乱套了。

有类型系统，编译器就像交警，帮你指挥每一条"数据"该走哪条路。
}

// 整数类型

整数（Integer） = 没有小数点的数字：`0`、`42`、`-100`、`2147483647`。

先上代码，手动跑一下感受一下：

\run[cpp]{
#include <iostream>
using namespace std;

int main() {
    int age = 19;              // 最常用的整数类型
    int temperature = -5;      // 可以是负数
    int score = 0;             // 可以是0

    cout << "年龄: " << age << endl;
    cout << "温度: " << temperature << endl;
    cout << "分数: " << score << endl;

    // 试试改改这些值，比如把 age 改成 999999
    // 观察输出有什么变化
    return 0;
}
}

\note{**`using namespace std;` 是什么？** 上节课我们写 `std::cout`，那个 `std::` 就是命名空间（Namespace）前缀。加了这行之后，你可以直接写 `cout` 而不用每次都写 `std::cout`。在学习阶段为了少打字，我们加上这行。}

C++有好几种整数类型，它们的区别在于**能存多大的数**：

\table{
类型 | 大小（通常） | 能存的范围 | 什么时候用
---
short | 2字节 | -32,768 ~ 32,767 | 很少用
int | 4字节 | 约 -21亿 ~ 21亿 | **最常用！** 绝大多数场景
long | 4或8字节 | 至少和int一样大 | 需要更大范围时
long long | 8字节 | 约 -922京 ~ 922京 | 超大数字
}

\diagram[compare]{
left_title: 各整数类型的"箱子"大小
left: [short: 2字节] [int: 4字节] [long long: 8字节]
right_title: 能装多大的数
right: [short: ±3万] [int: ±21亿] [long long: ±922京]
highlight: int
}

\tip{**日常原则：整数就用 `int`。** 只有当你确实需要非常大的数字（超过21亿）时才考虑 `long long`。游戏中的生命值、分数、坐标等用 `int` 绑绑有余。}

// 浮点数（小数）

浮点数（Floating Point） = 有小数点的数字：`3.14`、`0.5`、`-2.718`、`100.0`。

\table{
类型 | 大小 | 精度 | 什么时候用
---
float | 4字节 | 约7位有效数字 | 对精度要求不高时（游戏坐标）
double | 8字节 | 约15位有效数字 | **默认选择！** 日常计算用它
}

\tip{记忆口诀：**double = double精度**（双精度），所以它比 float（单精度）精确一倍，日常直接用 double 就对了。}

现在来看一个让所有初学者都大吃一惊的经典坑——浮点精度问题：

\run[cpp]{
#include <iostream>
using namespace std;

int main() {
    float pi_f = 3.14f;         // 单精度，注意末尾的 f
    double pi_d = 3.14159265;   // 双精度（默认）

    cout << "float pi:  " << pi_f << endl;
    cout << "double pi: " << pi_d << endl;

    // === 经典翻车现场 ===
    double a = 0.1 + 0.2;
    cout << "\n0.1 + 0.2 = " << a << endl;

    // 试试直接判等：
    if (a == 0.3) {
        cout << "等于 0.3！" << endl;
    } else {
        cout << "居然不等于 0.3！？？" << endl;
    }

    // 正确做法——判断差值是否足够小：
    if (a - 0.3 < 0.0001 && a - 0.3 > -0.0001) {
        cout << "用差值比较：确实约等于 0.3" << endl;
    }

    return 0;
}
}

\warning{**浮点数有精度问题！** `0.1 + 0.2` 在计算机里不等于 `0.3`！这是所有编程语言的通病（跟二进制表示有关），所以**永远不要用 `==` 直接比较两个浮点数是否相等**。}

\collapse[🔍 为什么 0.1 + 0.2 ≠ 0.3？]{
类比一下：你试过用分数精确表示 1/3 的十进制吗？结果是 0.333333... 永远写不完。

计算机用**二进制**（0和1）存小数也有类似问题——0.1 用二进制表示是一个无限循环小数！就像 1/3 的十进制写不完一样，计算机只能截断到有限位来存储，这就造成了微小的误差。

所以 `0.1 + 0.2` 实际上存的是约 `0.30000000000000004`，与精确的 `0.3` 差了一丁点。
}

// 字符（char）与 ASCII

`char`（Character）存储**单个字符**：一个字母、一个数字字符、一个标点符号。

\code[cpp]{
char grade = 'A';          // 用单引号！
char digit = '7';          // 这是字符 '7'，不是数字 7
char newline = '\n';       // 特殊字符：换行
}

\danger{**单引号 vs 双引号——别搞混了：**
- `'A'` → char（字符类型），单个字符，用**单引号**
- `"A"` → 字符串（string），用**双引号**
这两个在 C++ 眼里是**完全不同**的东西！}

字符的本质其实是**数字**。每个字符都对应一个数字编码，叫 **ASCII码（American Standard Code for Information Interchange）**。比如 `'A'` 对应 65，`'a'` 对应 97，`'0'` 对应 48。

所以你甚至可以对字符做数学运算——跑跑下面的代码试试：

\run[cpp]{
#include <iostream>
using namespace std;

int main() {
    char ch = 'A';
    cout << "字符: " << ch << endl;
    cout << "ASCII码: " << (int)ch << endl;      // 输出 65
    cout << "下一个字符: " << (char)(ch + 1) << endl;  // 输出 B

    // 小写字母和大写字母差32
    char lower = 'a';
    char upper = lower - 32;   // 'a'(97) - 32 = 65 = 'A'
    cout << lower << " 转大写 = " << upper << endl;

    // 字符 '0' 到 '9' 也是连续的
    char digit = '5';
    int num = digit - '0';    // '5'(53) - '0'(48) = 5
    cout << "字符'" << digit << "' 转数字 = " << num << endl;

    return 0;
}
}

\collapse[📋 常用 ASCII 码速查]{
- `'0'` ~ `'9'` → 48 ~ 57（连续，所以 `ch - '0'` 可得数字值）
- `'A'` ~ `'Z'` → 65 ~ 90（连续，大写字母）
- `'a'` ~ `'z'` → 97 ~ 122（连续，小写字母）
- 大写转小写：`ch + 32`；小写转大写：`ch - 32`
- 空格 `' '` → 32，换行 `'\n'` → 10
}

// 布尔值（bool）

`bool`（Boolean）是最简单的类型，只有两个值：`true`（真/是）或 `false`（假/否）。

\code[cpp]{
bool isAlive = true;       // 角色是否存活
bool gameOver = false;     // 游戏是否结束
bool hasKey = true;        // 是否拥有钥匙
}

在输出时，`true` 显示为 `1`，`false` 显示为 `0`。

\tip{**`bool` 在游戏开发中极其常用：** 角色是否存活？门有没有打开？任务完成了吗？按钮按下了吗？——这些"是或否"的问题都用 `bool`。虽然它只存 `true`/`false`，却是逻辑判断的基石。}

// sizeof——查看类型占多少字节

不确定某个类型占多少内存？用 `sizeof` 运算符（Operator）一查便知：

\run[cpp]{
#include <iostream>
using namespace std;

int main() {
    cout << "===== 各数据类型的大小 =====" << endl;
    cout << "bool:      " << sizeof(bool)      << " 字节" << endl;
    cout << "char:      " << sizeof(char)      << " 字节" << endl;
    cout << "short:     " << sizeof(short)     << " 字节" << endl;
    cout << "int:       " << sizeof(int)       << " 字节" << endl;
    cout << "long:      " << sizeof(long)      << " 字节" << endl;
    cout << "long long: " << sizeof(long long) << " 字节" << endl;
    cout << "float:     " << sizeof(float)     << " 字节" << endl;
    cout << "double:    " << sizeof(double)    << " 字节" << endl;

    // sizeof 也能用在变量上
    int x = 42;
    cout << "\n变量 x 占: " << sizeof(x) << " 字节" << endl;

    return 0;
}
}

\note{**1字节（Byte） = 8位（Bit）。** 1位能存 0 或 1，所以 1 字节能表示 2⁸ = 256 种不同的值。`int` 是 4 字节 = 32 位 = 能表示约 42 亿种值。其中一半给负数，所以范围大约是 -21亿 到 +21亿。}

\diagram[memory]{
stack:
  [1 byte]  bool b = true      (00000001)
  [1 byte]  char c = 'A'       (01000001 = 65)
  [4 bytes] int n = 42         (00000000 00000000 00000000 00101010)
  [8 bytes] double d = 3.14    (浮点编码，8字节)
}

// 有符号和无符号

默认情况下，整数类型是**有符号的（signed）**，即可以存正数也可以存负数。如果你确定一个值**永远不会是负数**，可以用 `unsigned`（无符号）：

\code[cpp]{
unsigned int health = 100;   // 血量不会是负数
unsigned int count = 0;      // 计数器不会是负数

// unsigned int 的范围：0 ~ 约42亿（把负数的空间全给了正数）
}

\tabs{
\tab[signed（有符号）]{
**范围：** -2,147,483,648 ~ 2,147,483,647（约 -21亿 ~ +21亿）

把 4 字节的 32 位空间**一分为二**：一半给负数，一半给正数和零。

**用途：** 温度、坐标、增减量——任何可能为负的值。

**默认就是 signed，不用特意写。**
}
\tab[unsigned（无符号）]{
**范围：** 0 ~ 4,294,967,295（约42亿）

把 4 字节的 32 位空间**全部**给非负数。

**用途：** 数组下标、像素颜色值、计数器——确定不会为负的值。

**⚠ 危险：** 减到 0 以下会溢出成超大正数！
}
}

\danger{**unsigned 的致命陷阱：** 如果 unsigned 的值减到 0 以下，不会变成 -1，而是"绕一圈"变成一个巨大的正数！这是游戏开发中经典的 bug 来源。}

\code[cpp]{
unsigned int x = 0;
x = x - 1;  // 你以为变成 -1？不！
// x 变成了 4294967295（约42亿）!!!
}

// 类型溢出——新手大坑

每种类型能存储的数值有上限。**超过上限就会溢出（Overflow）**——就像汽车里程表转到 999999 之后又变回 000000 一样。

\steps{
\step[看溢出前]{
`short` 类型的最大值是 32767。当它存的正好是 32767 时，一切正常。

\code[cpp]{
short s = 32767;  // short 的极限值
cout << s << endl; // 输出 32767，没问题
}
}
\step[触发溢出]{
给它 +1，超过了 `short` 能存的上限——结果并不是 32768，而是绕到最小值！

\code[cpp]{
short s = 32767;
s = s + 1;         // 溢出！
cout << s << endl;  // 输出 -32768 !!!
}
}
\step[理解原因]{
这就像时钟到了 12 点再拨一下不是 13 点，而是回到 1 点。数据在内存中循环：最大值 +1 = 最小值。

**这就是为什么有些老游戏里分数突然变成负数——溢出了！**
}
}

\danger{**有符号整数溢出在 C++ 中是未定义行为（Undefined Behavior, UB）。** 编译器可以做任何事情——结果不可预测。所以一定要选择足够大的类型来存你的数据。}

/// 难点易错点分析

\danger{**5个最常见的数据类型错误：**

**1. 整数除法丢失小数**
`int a = 7 / 2;` → a 等于 3（不是 3.5！两个 `int` 相除结果还是 `int`，小数部分直接砍掉）
修复：`double a = 7.0 / 2.0;` → a = 3.5

**2. float 字面量忘加 f**
`float x = 3.14;` → 编译器可能警告（`3.14` 默认是 `double` 类型）
修复：`float x = 3.14f;`

**3. 用 `==` 比较浮点数**
`if (0.1 + 0.2 == 0.3)` → 几乎永远不会为 true！用差值的绝对值是否小于一个很小的数来比较。

**4. char 和 int 混淆**
`char c = 65;` → c 存的是字符 `'A'`，不是数字 65
`int n = 'A';` → n 存的是数字 65，不是字符 A

**5. unsigned 减到负数变超大正数**
这是游戏开发中经典 bug 源头。不确定值会不会变负？那就老老实实用 `int`。}

// 自测区

\quiz[choice]{
Q: 下面这行代码执行后，`result` 的值是多少？\n`int result = 7 / 2;`
A: 3.5
B: 3
C: 4
D: 编译错误
ANS: B
EXP: 两个 `int` 相除，结果还是 `int`，小数部分直接被截断（不是四舍五入），所以 `7 / 2 = 3`。如果想得到 3.5，需要写成 `7.0 / 2.0`（让至少一个操作数变成浮点数）。
}

\quiz[tf]{
Q: `float x = 0.1 + 0.2;` 之后，`x == 0.3` 的结果是 `true`。
ANS: F
EXP: 浮点数在计算机中以二进制存储，`0.1` 和 `0.2` 都无法被精确表示，相加后的结果是约 `0.30000000000000004`，与精确的 `0.3` 有微小差异，所以 `==` 比较结果为 `false`。比较浮点数应该用差值法。
}

\quiz[choice]{
Q: `unsigned int x = 0; x = x - 1;` 执行之后，`x` 的值是什么？
A: -1
B: 0
C: 编译错误
D: 4294967295（约42亿）
ANS: D
EXP: `unsigned int` 不能存负数。当 0 减 1 时发生下溢（underflow），值会"绕一圈"到 `unsigned int` 的最大值 4294967295。这是无符号整数的特性，不是 bug，但经常被程序员误用而产生 bug。
}

// 🟡 进阶题：类型侦探

**任务：** 下面的代码有若干处类型相关的 bug，找出并修复它们。

\run[cpp]{
#include <iostream>
using namespace std;

int main() {
    // Bug 1: 想计算平均分，但结果不对
    int math = 95;
    int english = 88;
    int average = (math + english) / 2;
    cout << "平均分: " << average << endl;
    // 期望输出 91.5，实际输出是？为什么？怎么修？

    // Bug 2: 想存钱的金额，但类型选的不对
    int price = 9.99;
    cout << "价格: " << price << endl;
    // 输出是什么？为什么？怎么修？

    // Bug 3: 想输出字符 A，但写法有问题
    char letter = "A";
    cout << "字母: " << letter << endl;
    // 这行能编译通过吗？为什么？怎么修？

    return 0;
}
}

\collapse[🔑 进阶题参考思路]{
**Bug 1：** `(95 + 88) / 2` 全是 int 运算，结果为 91 而非 91.5。修复：把 `average` 改成 `double`，并让除法中有一个浮点数，例如 `/ 2.0`。

**Bug 2：** `int` 存不了小数，`9.99` 会被截断为 `9`。修复：改成 `double price = 9.99;`。

**Bug 3：** `"A"` 是双引号——这是字符串（string），不是字符（char）。修复：改成单引号 `'A'`。
}
