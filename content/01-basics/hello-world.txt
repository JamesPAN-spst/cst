/// 简介

上节课你已经成功运行了 `Hello, World!`。但那是照抄的——你可能还不知道每一行在干什么。这节课我们**逐字逐句**拆解那5行代码，让你彻底理解C++程序的基本结构。

读完这节，你就能自己写出各种输出程序，不再只是"复制粘贴别人的代码"了。

/// 知识点详解

// 先看完整代码

别急着逐行分析，我们先把完整代码跑一遍，看看效果：

\run[cpp]{
#include <iostream>

int main() {
    std::cout << "Hello, World!" << std::endl;
    return 0;
}
}

点击"▶ 运行"，你应该看到屏幕上输出了 `Hello, World!`。试试把双引号里的文字改成别的——比如你的名字——然后再运行一次。

一共5行有效代码（加上空行是6行）。现在我们逐行拆解。

// 逐行拆解

\steps{
\step[第1行：#include <iostream>]{
\code[cpp]{
#include <iostream>
}

**类比：** 你要做一道菜，但你家厨房没有锅。所以你先去隔壁借了一口锅回来。

`#include <iostream>` 就是"借工具"。它告诉编译器："我需要用输入输出功能（在屏幕上显示文字），请帮我把这个工具包加载进来。"

拆解这行：
- `#include` — "包含"（Include）的意思，把别人写好的工具导入你的代码
- `<iostream>` — 一个工具包的名字。`io` = input/output（输入/输出），`stream` = 流（数据流动的意思）

\note{**不写这行会怎样？** 编译器会报错，说"我不认识 `cout` 这个东西"。因为 `cout`（用来在屏幕上输出文字的工具）就定义在 `iostream` 这个工具包里。}
}
\step[第2行：空行]{
空行在C++中**没有任何意义**，纯粹是为了让代码看起来整齐。你可以加一百个空行，也可以一个都不加——程序运行结果一模一样。
}
\step[第3行：int main() {]{
\code[cpp]{
int main() {
}

这是整个程序**最重要的一行**。

**类比：** 一本书有很多章节，但你翻开书一定从第一章开始读。`main()` 就是程序的"第一章"——电脑运行你的程序时，第一个执行的地方。

拆解：
- `int` — 这个函数结束后会**返回一个整数**（后面解释）
- `main` — 函数的名字。这个名字是**固定的**，C++规定程序的入口必须叫 `main`，不能叫别的
- `()` — 括号里放"输入参数"（现在是空的，表示不需要任何输入）
- `{` — 大括号开始，表示"函数体开始了，下面的代码属于main函数"

\danger{**`main` 不能改名！** 如果你写成 `int Main()` 或 `int start()`，编译器会说"找不到main函数"，程序无法运行。大小写也必须完全一致。}
}
\step[第4行：std::cout << "Hello, World!" << std::endl;]{
\code[cpp]{
    std::cout << "Hello, World!" << std::endl;
}

这是实际干活的代码——在屏幕上显示文字。

拆解：
- `std::cout` — "标准输出"（Standard Console Output）。`std` 是一个**命名空间**（Namespace，你可以理解为"工具箱的名字"），`cout` 是里面的一个工具。双冒号 `::` 的意思是"从std这个工具箱里取出cout"
- `<<` — "往左边送"。把右边的内容送给左边的 `cout`，让它显示出来
- `"Hello, World!"` — 双引号里的是**字符串**（String，一段文字）。双引号告诉编译器："这不是代码，这是一段要原样输出的文字"
- `<< std::endl` — `endl` = end line（结束这一行），相当于按了一下回车键，光标移到下一行
- `;` — **分号！** C++中每一条语句（Statement）都必须以分号结尾，就像中文的句号

\tip{**一个更简单的理解方式：** 把 `<<` 想象成一个"传送带"。你把东西一个一个放上去，`cout` 会按顺序显示它们：
`cout << 东西1 << 东西2 << 东西3;`}

\warning{**最常见的新手错误：忘记写分号。** 编译器会报错，但错误信息经常指向**下一行**而不是真正出错的行。所以如果某一行报错但你看不出问题，检查一下**上一行**的末尾是不是少了分号。}
}
\step[第5行：return 0;]{
\code[cpp]{
    return 0;
}

**类比：** 你完成了一个任务，向老板汇报"任务完成，一切正常"。

`return 0` 就是main函数向操作系统汇报："程序正常结束了。"

- `return` — "返回"，结束这个函数
- `0` — 在编程的惯例中，返回0表示"成功/正常"。如果返回其他数字（如1、-1），表示"出错了"

\note{**实际上，现代C++中 `return 0;` 可以省略。** 编译器会自动在 `main` 函数末尾加上 `return 0;`。但写出来是个好习惯，特别是你在学习阶段。}
}
\step[第6行：}]{
\code[cpp]{
}
}

右大括号，与第3行的左大括号 `{` 配对，表示"main函数到此结束"。

\danger{**大括号必须配对！** 有 `{` 就必须有对应的 `}`。VS Code会自动高亮匹配的括号——把光标放在一个括号旁边，它的配对括号会被高亮。}
}
}

\collapse[关于 using namespace std]{
你可能在网上看到别人的代码开头写了这么一行：

\code[cpp]{
using namespace std;
}

加了这行之后，就不用每次都写 `std::cout`，可以直接写 `cout`。相当于"把整个std工具箱里的东西都倒出来摆在桌上"——方便，但桌子会变乱。

**我们的建议：** 初学阶段可以用，但养成写 `std::` 的习惯更好。大项目中 `using namespace std;` 可能导致名字冲突（两个工具箱里有同名工具），很多公司的编码规范明确禁止使用它。

本课程的示例代码统一使用 `std::` 前缀写法。
}

// 注释——给自己写笔记

代码中可以写"注释"（Comment）——编译器会**完全忽略**注释的内容，它只是给人看的笔记。

C++有两种注释方式：

\list{
- `//` 单行注释——两个斜杠后面直到行末的内容都被忽略
- `/* ... */` 多行注释——可以跨越多行，适合写大段说明
}

试试在下面的代码中加上你自己的注释，然后运行看看——注释不会影响输出结果：

\run[cpp]{
#include <iostream>

int main() {
    // 这是单行注释，编译器会完全忽略这行
    std::cout << "Hello!" << std::endl;  // 行末也能写注释

    /*
       这是多行注释
       可以跨越好几行
       适合写大段说明
    */
    std::cout << "Comments don't affect output!" << std::endl;
    return 0;
}
}

\tip{**养成写注释的习惯。** 你今天写的代码，一周后可能就忘了为什么这么写。注释是给"未来的你"留的备忘录。}

// 缩进——代码的"格式"

你可能注意到第4、5行前面有空格。这叫**缩进**（Indentation）。

\code[cpp]{
int main() {
    std::cout << "Hello!";  // ← 这行有4个空格的缩进
    return 0;               // ← 这行也有
}
}

缩进的作用是**让代码更好读**。它告诉你"这些代码属于main函数内部"。

C++不强制缩进（不缩进也能运行），但**所有程序员都用缩进**。VS Code会自动帮你缩进——按回车后它会自动加上合适的空格。

\collapse[endl 和 \\n 有什么区别？]{
你可能在别的教程里看到用 `\n` 换行，而不是 `std::endl`：

\code[cpp]{
std::cout << "Hello" << std::endl;  // 方式1：用 endl
std::cout << "Hello\n";            // 方式2：用 \n
}

两者都能换行，但有一个区别：`std::endl` 除了换行还会**刷新缓冲区**（Flush，强制把数据立刻输出到屏幕），而 `\n` 只换行不刷新。

**初学阶段两个都行。** 用哪个看个人习惯。后面学到性能优化时会再讨论。
}

// 从代码到运行——完整流程

你写好了 `.cpp` 文件之后，电脑并不能直接运行它——电脑只懂0和1。你的代码需要经过**编译**（Compilation）才能变成可执行文件。

\diagram[flow]{
你写代码(.cpp文件) -> 预处理(展开#include) -> 编译(翻译成机器码) -> 链接(合并所有部分) -> 可执行文件(.exe)
}

用命令行编译运行的完整步骤：

\code[bash]{
g++ hello.cpp -o hello   # 编译：把 hello.cpp 翻译成 hello.exe
./hello                  # 运行：执行生成的可执行文件
}

- 如果**编译报错** → 你的代码有语法错误，回去检查代码
- 如果**编译成功但运行不对** → 逻辑有问题，需要调试
- 如果**找不到 g++** → 开发环境没装好，回去看 env-setup 那节

\note{**VS Code 用户不用手动敲命令。** 配置好之后，按 `Ctrl+Shift+B` 编译，按 `F5` 运行，IDE 会自动帮你执行上面的步骤。}

/// 难点易错点分析

\danger{**新手最常犯的5个错误：**

**1. 忘记分号**
`cout << "hi"` → 错！应该是 `cout << "hi";`

**2. 用中文标点**
`cout << "你好"；` → 错！分号是中文的。必须用英文 `;`
同理，双引号必须是英文的 `""`，不能是中文的 `""`

**3. 大小写写错**
`Int main()` → 错！是 `int`（全小写）
`Cout` → 错！是 `cout`（全小写）
`#Include` → 错！是 `#include`（全小写）

**4. 大括号不配对**
少写或多写一个 `{` 或 `}`，编译器会报一堆看不懂的错误

**5. 字符串没有用双引号包裹**
`cout << Hello;` → 错！编译器以为Hello是一个变量名
`cout << "Hello";` → 对！双引号告诉编译器这是一段文字}

来做几道题检验一下你学到了什么：

\quiz[choice]{
Q: `#include <iostream>` 这行代码的作用是什么？
A: 定义了一个叫 iostream 的变量
B: 导入输入输出功能的工具包，让我们能使用 `cout` 等工具
C: 在屏幕上输出 iostream 这个单词
D: 告诉编译器程序从这里开始执行
ANS: B
EXP: `#include` 是"包含"指令，它把 `<iostream>` 这个**头文件**（Header File）中定义的输入输出工具（如 `cout`、`cin`、`endl`）导入到你的代码中。没有这行，编译器就不认识 `cout`。
}

\quiz[choice]{
Q: 以下代码哪里有错误？
`std::cout << "Game Over!" << std::endl`
`return 0;`
A: `#include` 不应该有 `<>`
B: `main()` 前面不应该写 `int`
C: 第一行末尾少了分号 `;`
D: `return 0` 应该写成 `return 1`
ANS: C
EXP: C++中每条语句都必须以分号 `;` 结尾。`std::cout << "Game Over!" << std::endl` 这行末尾缺少分号，编译器会在**下一行**（`return 0;`）附近报错——这就是为什么新手经常找不到真正的错误位置。
}

\quiz[tf]{
Q: 在C++中，`main` 函数的名字可以改成 `start` 或 `begin`，只要统一就行。
ANS: F
EXP: `main` 是C++标准规定的**固定名称**，操作系统启动程序时会自动寻找叫 `main` 的函数作为入口。改成任何其他名字（`start`、`Main`、`begin`）都会导致链接器报错"找不到 main 函数"。
}
