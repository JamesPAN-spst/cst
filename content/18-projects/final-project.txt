/// 简介

这是整个 C++ 技能树的 **毕业设计（Capstone Project）**。你将整合全部 18 章的知识，从零构建一个包含物理、渲染、音频、脚本、工具链的 **完整 2D 游戏引擎**，并用这个引擎制作一个可玩的 DEMO 游戏。

这不是一个"做着玩"的小练习——它是一个真正的工程项目，要求你做出架构决策、管理子系统依赖、处理跨模块集成问题。完成后，你将拥有一个可以写进简历的引擎项目，以及对 C++ 工程能力的全面验证。

/// 项目需求

**项目名称：** MiniEngine2D — 完整的 2D 游戏引擎 + DEMO 游戏

\list{
- 构建一个分层架构的 2D 游戏引擎
- 引擎包含：物理、渲染、音频、脚本、资源管理、工具链
- 使用该引擎制作一个可玩的 DEMO 游戏（类型自选）
- DEMO 必须包含：至少 2 个关卡/场景、存档、音效、基本 AI
}

**DEMO 游戏参考方向（三选一或自拟）：**

\tabs{
\tab[方向A: 平台跳跃]{
经典 2D 平台跳跃游戏。物理引擎驱动角色运动和碰撞，多个关卡，收集物品，敌人 AI（巡逻+追踪），粒子特效（跳跃尘土、死亡效果）。
}
\tab[方向B: 顶视角射击]{
双摇杆射击（或键鼠控制）。波次刷怪、不同敌人类型、弹道物理、道具掉落、分数系统。物理引擎处理弹丸碰撞和击退效果。
}
\tab[方向C: 物理解谜]{
用物理交互解谜：推箱子、杠杆、绳索（Verlet）、弹簧。物理引擎是核心玩法。关卡由 Lua 脚本驱动。需要关卡编辑器（简易版）导出关卡数据。
}
}

场景管理 | Ch.15 | 场景加载/卸载/切换
事件系统 | Ch.15 | 类型安全的 publish/subscribe

/// 实现提示

\steps{
\step[M1 — 项目骨架（第1周）]{
CMake 项目结构、SDL2 窗口创建、游戏循环（固定步长）、输入处理、数学库。此阶段结束时：能打开窗口、显示纯色背景、响应键盘输入、以固定 60Hz 步进。
}
\step[M2 — 核心引擎（第2周）]{
精灵渲染器（批处理）、纹理加载、物理引擎（圆+AABB碰撞+响应）、音频系统（miniaudio）。此阶段结束时：能在屏幕上画精灵、有碰撞反弹的物体、能播放音效。
}
\step[M3 — 引擎框架（第3周）]{
子系统管理（Engine 类）、资源管理器（缓存+热重载）、事件系统、场景管理。Lua 集成（sol3）——绑定游戏对象。此阶段结束时：引擎架构完整，可以用 Lua 脚本控制游戏逻辑。
}
\step[M4 — DEMO 游戏（第4周）]{
用引擎制作 DEMO：关卡设计、敌人 AI（Lua 编写）、粒子特效、UI（分数/血条）、存档系统（JSON 序列化）。ImGui 调试面板。此阶段结束时：DEMO 完整可玩。
}
\step[M5 — 打磨与文档（第5周）]{
性能优化（profiler 定位瓶颈）、内存检查（ASan）、Bug 修复、README 文档、构建说明。此阶段结束时：项目可提交。
}
}

\warning{**不要试图一开始就设计"完美架构"。** 先让最简版本跑起来（窗口 + 一个精灵 + 碰撞），然后迭代重构。过早抽象会拖慢进度。}

\tip{**依赖管理推荐方案：** 用 CMake FetchContent 拉取 SDL2、sol2、miniaudio、stb_image、nlohmann_json、Dear ImGui。避免手动管理第三方库路径。}
