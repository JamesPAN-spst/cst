/// 简介

上一节我们学习了栈（Stack）与堆（Heap）的区别——栈上的内存由编译器自动管理，而堆上的内存需要程序员手动申请和释放。那么，C++ 中向堆申请/归还内存的"正式入口"是什么？答案就是 `new` 和 `delete`。

你也许会问："现代 C++ 不是推荐智能指针吗？学裸 `new`/`delete` 还有意义吗？"有意义——而且非常重要。智能指针的底层就建立在 `new`/`delete` 之上；内存池（Memory Pool）、自定义分配器（Allocator）、游戏引擎中的对象工厂……这些高性能场景都离不开对 `operator new`/`operator delete` 的精确控制。只有理解了底层机制，你才能在需要时突破智能指针的"默认行为"，写出更快、更安全的代码。

本节将从最基本的 `new`/`delete` 表达式出发，逐步深入到 placement new、对齐分配、全局和类级别的 `operator new` 重载，最终理解现代 C++ 为什么推荐 `make_unique`/`make_shared`。

/// 知识点详解

// new 表达式的两步操作

当你写下 `Widget* w = new Widget(42);` 时，编译器实际上做了**两件事**：

**第一步——分配原始内存（Raw Memory Allocation）：** 调用 `operator new(sizeof(Widget))` 向堆申请一块足够大的原始内存。这一步类似 C 语言的 `malloc`，返回的是 `void*`，还没有任何 `Widget` 对象。

**第二步——在原始内存上构造对象（Construction）：** 在刚分配的内存地址上调用 `Widget` 的构造函数（Constructor），把这块"生肉"变成一个真正的 `Widget` 对象。

\code[cpp]{
// new 表达式的背后
Widget* w = new Widget(42);

// 编译器展开后大致等价于：
void* raw = operator new(sizeof(Widget));   // 第一步：分配原始内存
Widget* w2 = new (raw) Widget(42);          // 第二步：placement new 调用构造函数
}

同理，`delete w;` 也做两件事，但**顺序相反**：

**第一步——析构对象（Destruction）：** 调用 `w->~Widget()` 执行析构函数。

**第二步——释放原始内存（Deallocation）：** 调用 `operator delete(w)` 把内存还给系统。

\code[cpp]{
delete w;

// 等价于：
w->~Widget();           // 先析构
operator delete(w);     // 再释放
}

\note{**关键区分：** `new` **表达式**会"分配+构造"；而 `operator new` **函数**只做"分配"。前者是语言关键字驱动的完整流程，后者是一个可以被重载的普通函数。两者名字相似，但含义不同。}

\diagram[flow]{
new Widget(42) -> operator new(sizeof(Widget)) -> 原始内存 void* -> 调用构造函数 Widget(42) -> 完整对象 Widget*
}

// new[] 与 delete[] 数组版本

当你需要一次分配多个对象时，使用数组版本 `new[]`。它调用 `operator new[](总字节数)` 分配内存，然后依次对每个元素调用构造函数。

\code[cpp]{
int* arr = new int[100]{};      // 分配 100 个 int，值初始化（Value-Initialization）为 0
delete[] arr;                    // 用 delete[] 释放

Widget* ws = new Widget[5];     // 分配 5 个 Widget，调用默认构造函数
delete[] ws;                    // 逆序析构 ws[4]→ws[0]，再释放内存
}

铁律：**`new` 搭配 `delete`，`new[]` 搭配 `delete[]`，混用是未定义行为（Undefined Behavior, UB）。**

\danger{**混用 new/delete 与 new[]/delete[] 是最常见的内存 Bug。** `new[]` 通常在头部存储元素个数。用 `delete` 代替 `delete[]` 会只析构第一个元素（资源泄漏），且传给 `operator delete` 的指针可能偏移（堆损坏）。}

\run[cpp]{
#include <iostream>

struct Loud {
    int id;
    Loud(int i) : id(i) { std::cout << "构造 Loud #" << id << "\n"; }
    ~Loud()              { std::cout << "析构 Loud #" << id << "\n"; }
};

int main() {
    std::cout << "=== new[] + delete[] (正确) ===\n";
    Loud* arr = new Loud[3]{ {1}, {2}, {3} };
    delete[] arr;  // 正确：逆序析构 3→2→1

    std::cout << "\n=== 单个 new + delete (正确) ===\n";
    Loud* single = new Loud(99);
    delete single;

    // 千万不要写：delete arr; 或 delete[] single; —— 这是 UB！
    return 0;
}
}

// placement new（定位 new）

有时候你已经有一块内存（内存池、栈上缓冲区、共享内存），只需在上面**构造**对象而不重新分配。这就是 **placement new（定位 new）**：跳过分配，只执行构造。

\code[cpp]{
#include <new>  // placement new 需要这个头文件

// 在栈上的缓冲区里构造一个 Widget
alignas(Widget) unsigned char buffer[sizeof(Widget)];
Widget* w = new (buffer) Widget(42);  // 不分配内存！只调用 Widget(42) 构造函数

// 使用 w ...

// 用完后必须手动调用析构函数——不能用 delete！
// 因为这块内存不是 operator new 分配的
w->~Widget();
}

placement new 的典型应用：**内存池**预分配大块内存避免系统调用；**容器内部**如 `std::vector` 在缓冲区上构造元素；**共享内存**在 `mmap` 区域上构造对象；**延迟初始化**先分配后择时构造。

\warning{**placement new 构造的对象必须手动调用析构函数。** 这是 C++ 中极少数需要显式析构的场景。`new` 分配的对象由 `delete` 负责析构，不要手动调。}

\diagram[memory]{
stack:
  [buffer] Widget (placement new)
heap:
  (无堆分配)
note: placement new 在已有内存上构造，不涉及堆
}

// nothrow new

默认的 `new` 在分配失败时抛出 `std::bad_alloc` 异常（Exception）。如果不想用异常（嵌入式/实时系统），可用 **nothrow new**：失败返回 `nullptr`。

\code[cpp]{
#include <new>
int* p = new (std::nothrow) int[1000000000];  // 失败返回 nullptr
if (!p) { std::cerr << "内存不足！\n"; }
}

\danger{**默认 `new` 分配失败时抛 `std::bad_alloc`，不返回 `nullptr`！** 很多 C 转 C++ 的程序员习惯写 `int* p = new int; if (!p) ...`——这种检查永远不会触发，因为 `new` 要么成功要么抛异常。要想检查 `nullptr`，必须用 `new (std::nothrow)`。}

// set_new_handler：自定义分配失败策略

在 `new` 抛出 `std::bad_alloc` 之前，它会先调用当前安装的 **new-handler（分配失败处理函数）**。你可以用 `std::set_new_handler()` 设置自定义函数，在内存不足时尝试补救——释放缓存、写日志、或优雅终止。

\code[cpp]{
#include <new>
void my_new_handler() {
    // 尝试释放缓存内存...
    std::set_new_handler(nullptr);  // 移除 handler，下次失败直接抛 bad_alloc
}
int main() {
    std::set_new_handler(my_new_handler);
    // new 失败时循环调用 handler → 重试，直到成功或 handler 为 nullptr 时抛 bad_alloc
}
}

\note{`operator new` 在循环中调用 new-handler → 重试分配，直到成功或 handler 抛异常/被设为 `nullptr`。这让你有机会在内存紧张时做"紧急回收"。}

// 全局 operator new / operator delete 重载

`operator new` 和 `operator delete` 是普通函数，你可以**全局重载**它们来追踪所有动态分配、统计内存使用、或者替换默认分配器。

\run[cpp]{
#include <iostream>
#include <cstdlib>
#include <new>

// 全局重载 operator new —— 影响整个程序的所有 new 表达式
void* operator new(std::size_t size) {
    std::cout << "[alloc] " << size << " bytes\n";
    void* p = std::malloc(size);
    if (!p) throw std::bad_alloc{};
    return p;
}

// 全局重载 operator delete
void operator delete(void* p) noexcept {
    std::cout << "[free] at " << p << "\n";
    std::free(p);
}

// 带大小的 operator delete（C++14, sized deallocation）
void operator delete(void* p, std::size_t size) noexcept {
    std::cout << "[free] " << size << " bytes at " << p << "\n";
    std::free(p);
}

int main() {
    int* a = new int(42);
    delete a;

    std::cout << "---\n";

    double* arr = new double[3]{1.0, 2.0, 3.0};
    delete[] arr;

    return 0;
}
}

\warning{**全局重载 `operator new`/`delete` 影响整个程序（含标准库）。** 生产中要谨慎，通常只在调试/性能分析阶段使用。}

// 类特定的 operator new / operator delete

如果你只想控制某个类的分配行为而不影响全局，可以在类内部定义 `operator new` / `operator delete` 作为**静态成员函数**（Static Member Function）。编译器对该类的 `new`/`delete` 表达式会优先调用类版本。

\code[cpp]{
class PooledObject {
public:
    void* operator new(std::size_t size) {           // 隐式 static
        return pool.allocate(size);                   // 从内存池分配
    }
    void operator delete(void* p) noexcept {
        pool.deallocate(p);                           // 归还给内存池
    }
    static MemoryPool pool;
};
// new PooledObject → 调用 PooledObject::operator new
// new int(1)        → 不受影响，调用全局 operator new
}

\tip{类级别重载 `operator new`/`delete` 是实现**对象池（Object Pool）** 的经典手段。在游戏引擎中，频繁创建/销毁的小对象（子弹、粒子、事件）常用此技术避免碎片化和系统调用开销。}

\collapse[🤔 想一想：类的 operator new 可以被继承吗？]{
**可以。** 派生类继承基类的 `operator new`/`delete`。但派生类 `sizeof` 更大，固定大小内存池可能装不下。安全做法：检查 `size != sizeof(Base)` 时退回 `::operator new(size)`。
}

/// 新版特性（C++17/20/23）

// 对齐分配（Aligned Allocation, C++17）

C++17 之前，**过度对齐类型（Over-Aligned Type）** 只能用平台特定函数分配（`_aligned_malloc` / `posix_memalign`）。C++17 引入了对齐版本的 `operator new`，接受 `std::align_val_t` 参数，编译器自动为过度对齐类型选择：

\code[cpp]{
struct alignas(64) CacheLine { int data[16]; };  // 64 字节缓存行对齐

CacheLine* p = new CacheLine;  // C++17 自动调用对齐版本 operator new
delete p;                       // 自动调用对齐版本 operator delete

// 手动对齐分配
void* raw = operator new(1024, std::align_val_t{64});
operator delete(raw, std::align_val_t{64});
}

// sized deallocation（带大小的释放, C++14）

C++14 允许 `operator delete` 接受额外的 `std::size_t` 参数，告知释放函数内存大小，分配器可快速定位对应的大小类/内存池桶。

\code[cpp]{
void operator delete(void* p, std::size_t size) noexcept {
    // size = 当初分配的字节数，编译器自动传入
    std::free(p);
}
}

\note{GCC 需 `-fsized-deallocation` 选项启用。Clang/MSVC 默认启用。编译器优先调用带 size 的版本。}

// 为什么现代 C++ 推荐 make_unique / make_shared

理解了底层机制后，来看为什么现代 C++ **不推荐直接用裸 `new`/`delete`**：

\compare{
old: 裸 new 有三大问题——异常安全隐患（`shared_ptr<A>(new A)` 与 `shared_ptr<B>(new B)` 交叉求值时可能泄漏）；容易忘 delete 或 new/delete[] 不匹配；所有权语义不清晰。

new: `make_unique` (C++14) / `make_shared` (C++11) 原子地完成分配+构造+绑定，异常安全；自动管理生命周期，无需手动 delete；`make_unique<int[]>(100)` (C++20) 替代 `new int[100]`。
}

\tip{**经验法则：** 日常用 `make_unique`/`make_shared`；自定义内存管理用 `allocator` 或类级别 `operator new`；只在实现容器/内存池/分配器时才直接用 `operator new` + placement new。}

/// 工程实践

// 实际开发中的最佳实践

\list{
- **Rule of Zero：** 不直接管理内存的类不要写析构/拷贝/移动，让编译器默认版本配合智能指针
- **裸 new 立刻交智能指针接管：** `auto p = std::unique_ptr<Widget>(new Widget(args));`
- **重载 operator new/delete 必须配对：** new↔delete、new[]↔delete[]、对齐版本同理
- **全局重载仅用于调试：** 生产环境替换分配器请用 jemalloc / tcmalloc / mimalloc
- **placement new 后必须手动析构，** 这是唯一合法的显式析构调用场景
- **两步模式异常安全：** `operator new` + placement new 时用 try-catch 保护
}

\table{
场景 | 推荐做法 | 避免做法
---
单个动态对象 | `std::make_unique<T>(args)` | `new T(args)` 后手动 delete
动态数组 | `std::vector<T>` 或 `make_unique<T[]>(n)` | `new T[n]` 后手动 delete[]
需要共享所有权 | `std::make_shared<T>(args)` | `std::shared_ptr<T>(new T)` 裸 new 构造
自定义内存源 | 类级别 `operator new` + 内存池 | 全局替换 `operator new`
性能敏感小对象 | 对象池 + placement new | 每次都 `new`/`delete` 系统调用
}

/// 难点易错点分析

\danger{**异常安全与两步拆分：** `operator new` + placement new 拆开后，构造函数抛异常时你必须自己释放内存。安全写法：}

\code[cpp]{
void* mem = operator new(sizeof(Widget));
try {
    new (mem) Widget(dangerous_args);
} catch (...) {
    operator delete(mem);  // 手动释放
    throw;
}
}

\danger{**double delete（双重释放, Double Free）：** 对同一块内存调用两次 `delete` 是 UB，可导致崩溃或安全漏洞。缓解：`delete` 后置 `nullptr`；根治：用智能指针。}

\danger{**数组 operator new 的隐藏开销：** `new T[n]` 可能额外分配字节存储元素个数，因此 `operator new[]` 收到的 `size` 可能大于 `n * sizeof(T)`。}

\quiz[choice|单选]{
Q: 以下代码中哪一行存在未定义行为？
\code[cpp]{
int* a = new int(10);       // 行1
int* b = new int[5]{};      // 行2
delete[] a;                  // 行3
delete[] b;                  // 行4
}
A: 行1
B: 行2
C: 行3
D: 行4
ANS: C
EXP: `a` 由 `new int(10)`（非数组 new）分配，必须用 `delete a` 释放。`delete[] a` 是未定义行为——`new` 和 `delete[]` 不匹配。行4 是正确的，`b` 由 `new int[5]` 分配，用 `delete[] b` 释放。
}

\quiz[tf|判断]{
Q: `int* p = new int; if (!p) handleError();` 中的 `if` 分支永远不会执行。
ANS: T
EXP: 默认的 `new` 在分配失败时抛出 `std::bad_alloc` 异常，不返回 `nullptr`。因此 `p` 要么指向有效内存，要么程序因异常跳走——`if (!p)` 永远为 `false`。想检查 `nullptr` 必须用 `new (std::nothrow) int`。
}

\quiz[choice|单选]{
Q: placement new 构造的对象应该如何销毁？
A: 直接调用 `delete p;`
B: 调用 `operator delete(p);`
C: 手动调用析构函数 `p->~T();`
D: 不需要销毁，超出作用域自动释放
ANS: C
EXP: placement new 没有分配内存，只执行了构造。因此销毁时也只需要手动调用析构函数 `p->~T()`，不能用 `delete`（`delete` 会尝试释放不是 `operator new` 分配的内存，导致 UB）。底层内存由其原始所有者负责释放。
}

// 🟡 进阶题 2：类级别内存池

为 `Bullet` 类实现简化版对象池：预分配固定数组作为内存池，通过类级别 `operator new`/`delete` 管理。

\run[cpp]{
#include <iostream>
#include <new>

class Bullet {
public:
    float x, y, speed;
    Bullet(float px, float py, float spd) : x(px), y(py), speed(spd) {
        std::cout << "Bullet(" << x << "," << y << ")\n";
    }
    ~Bullet() { std::cout << "~Bullet\n"; }

    static constexpr int MAX = 8;
    static void* operator new(std::size_t size);
    static void operator delete(void* p) noexcept;
    /* TODO: 实现上面两个函数
       提示：static bool used[MAX] 标记槽位；
       static alignas(Bullet) unsigned char pool[MAX][sizeof(Bullet)]；
       new 找空闲槽返回地址，delete 标记释放，池满抛 bad_alloc */
private:
    static bool used[MAX];
    static alignas(Bullet) unsigned char pool[MAX][sizeof(Bullet)];
};
bool Bullet::used[MAX] = {};
alignas(Bullet) unsigned char Bullet::pool[MAX][sizeof(Bullet)] = {};

/* TODO: 实现 Bullet::operator new / delete */

int main() {
    Bullet* b1 = new Bullet(0,0,10);
    Bullet* b2 = new Bullet(1,2,15);
    delete b2;
    Bullet* b3 = new Bullet(5,6,25);  // 应复用 b2 的槽位
    delete b1; delete b3;
    return 0;
}

// 🔴 挑战题：实现 mini vector

用 `operator new` + placement new + 显式析构 + `operator delete` 实现简化版 `MiniVector<T>`。

**要求：** 实现 `push_back`、`pop_back`、`operator[]`、`size()`、`reserve()`。扩容翻倍（初始 4）。析构函数逆序析构元素后 `operator delete` 释放内存。异常安全：构造第 N 个失败时析构已构造的 0..N-1。用 AddressSanitizer（`g++ -fsanitize=address`）验证无泄漏。

\collapse[参考思路]{
\code[cpp]{
template<typename T>
class MiniVector {
    T* data_ = nullptr;
    size_t size_ = 0, capacity_ = 0;
    void grow() {
        size_t nc = capacity_ ? capacity_ * 2 : 4;
        T* nd = static_cast<T*>(operator new(nc * sizeof(T)));
        for (size_t i = 0; i < size_; ++i)
            new (nd + i) T(std::move(data_[i]));  // placement new 移动旧元素
        for (size_t i = size_; i > 0; --i)
            (data_ + i - 1)->~T();                // 析构旧元素
        operator delete(data_);                    // 释放旧内存
        data_ = nd; capacity_ = nc;
    }
public:
    void push_back(const T& v) { if (size_==capacity_) grow(); new(data_+size_) T(v); ++size_; }
    void pop_back() { --size_; (data_+size_)->~T(); }
    ~MiniVector() { for (size_t i=size_; i>0; --i) (data_+i-1)->~T(); operator delete(data_); }
};
}
四步分离：`operator new` 分配字节 → placement new 构造 → 显式析构 → `operator delete` 释放。
}
